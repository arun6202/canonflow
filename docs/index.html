Oracle-Level Strictness in F#

Symphony is a state-of-the-art data bridge that eliminates runtime data anomalies. By parsing native database constraints and elevating them directly into the F# compiler's type system, we guarantee data integrity at compile time.

What We Accomplished

SqlHydra Core Integration

We extracted the raw SqlHydra generator into the Symphony codebase. By natively patching the CLI architecture, we brought code-generation completely in-house without breaking standard cross-database workflows.

SQLite Constraint Parsing

Engineered a robust SQLite parser that queries the sqlite_master table, extracts raw SQL string definitions, and actively detects constraints such as CHECK ([Quantity]>0).

Refined Types AST Injection

Connected the parsed constraints directly to the Fantomas F# AST generator. SQLite constraints dynamically compile into explicitly typed structural predicates for flawless schema alignment.

The State-of-the-Art (SOTA) Approach

Instead of relying on fragile runtime reflection, attributes, or ORM interception logic, Symphony embeds validation strictly at the compiler level.

1

The Source Database

SQLite stores constraints loosely inside its schema definition. Historically, applications read this data directly into unconstrained primitives like int64 or decimal. 


CREATE TABLE [Order Details] (
    [OrderID] int NOT NULL,
    [UnitPrice] NUMERIC NOT NULL DEFAULT 0,
    CHECK ([UnitPrice]>=(0))
);
2

The Structural Interface

We utilize F# Structs to define statically resolved predicate filters. This design ensures that constraint verification executes with zero heap allocations. 


[<Struct>]
type GreaterThanOrEqualToZero =
    interface IPredicate<decimal> with 
        member _.Check(v) = v >= 0M

[<Struct>]
type Refined<'T, 'P when 'P :> IPredicate<'T> and 'P : struct> = 
    private { Value: 'T }
3

The Symphony Generator

During build time, the custom SqlHydra.Cli tool parses the SQL CHECK clause, matches it to the F# Predicate, and emits perfectly type-safe models using Fantomas. 


type ``Order Details`` =
    { OrderID: int64
      UnitPrice: Refined<decimal, GreaterThanOrEqualToZero> }

Result: If an application attempts to insert -10M into UnitPrice, it won't trigger a late database exception. It won't even compile unless it correctly passes through the Refined.create factory, guaranteeing invariant protection!
