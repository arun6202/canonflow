Idiomatic Refactoring Report

Showcasing the evolution to Stylish F#, Functional Domain Modeling, and Idiomatic PowerShell.

1. Primitive Obsession → Constrained Domains (F#)

Applying Functional Domain Modeling to make illegal states unrepresentable.

Before (C#-Style Records)
- type CustomerInfo = {
-     CustomerId: string
-     CompanyName: string
- }
- 
- // CustomerId could be "", null, or invalid.
- // Validation happens randomly across the codebase.
After (Stylish F# Types)
+ type CustomerId = private CustomerId of string
+ module CustomerId =
+     let create (s: string) = 
+         if String.IsNullOrWhiteSpace(s) then Error "Empty" 
+         else Ok (CustomerId (s.Trim()))
+
+ type CustomerInfo = { CustomerId: CustomerId }

2. Exception Throwing → Railway-Oriented Programming (F#)

Replacing hidden exceptions with explicit `Result` pipelines.

Before (Implicit Failures)
- let indexData (orders: OrderDocument list) =
-     // Throws if DB fails or client disconnects.
-     let client = new ElasticsearchClient()
-     let response = client.Bulk(...)
-     if not response.IsValid then
-         printfn "Warning: failed" // Hidden failure
After (Result.bind Pipelines)
+ let indexData (esUrl: string) (orders: list) : Result<unit, string> =
+     try
+         let client = new ElasticsearchClient()
+         // ...
+         if response.IsValidResponse then Ok ()
+         else Error $"Bulk failed: {resp.Debug}"
+     with ex -> Error $"Connection error: {ex.Message}"

3. Bash-Style Scripts → Advanced Functions (PowerShell)

Moving from raw scripts to pipeline-aware cmdlets with Strict Mode.

Before (Imperative Scripting)
- $northwindDir = "E:\github\northwind"
- Write-Host "Running App..."
- dotnet run
- Write-Host "Total orders: $($resp.hits.total.value)"
- // Fails silently if variable is typoed
After (Idiomatic Advanced Function)
+ [CmdletBinding()]
+ param( [ValidateNotNullOrEmpty()][string] $NorthwindDir )
+ Set-StrictMode -Version Latest
+ $ErrorActionPreference = "Stop"
+ 
+ Write-Information "Running App..."
+ [pscustomobject]@{ TotalOrders = $resp.hits.total.value }

SWOT Analysis of the Idiomatic Shift

Strengths
  • Compile-Time Safety: Invalid domain states (like negative IDs or empty strings) are literally impossible to compile.
  • Predictable Control Flow: `Result` makes failures explicit. No more hunting for caught exceptions.
  • Robust Orchestration: PowerShell Strict Mode catches silent variable typos immediately.
Weaknesses
  • Initial Boilerplate: Single-case DUs require helper modules (e.g., `CustomerId.value`, `CustomerId.create`).
  • Serialization Friction: External libraries (like Elasticsearch v9) struggle to serialize pure F# DUs, requiring explicit DTO boundaries.
  • Steeper Curve: Developers used to imperative C# loops may struggle with `fold` and `bind`.
Opportunities
  • Pure Unit Testing: Because business logic is isolated from I/O (Elasticsearch/SQLite), testing the core logic becomes trivial.
  • Self-Documenting Code: The `Domain.fs` file reads exactly like the business requirements.
  • Composability: Smaller functional pipelines can be easily reused and recombined.
Threats
  • Over-Engineering: Taking DDD too far on simple CRUD apps can slow velocity.
  • Ecosystem Mismatch: Heavy reliance on traditional C# OOP libraries may require writing many mapping DTOs (Data Transfer Objects).
  • PowerShell Versioning: Strict mode and advanced object syntax may break on legacy Windows PowerShell 5.1 environments.
